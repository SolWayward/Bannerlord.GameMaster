Query Commands

Advanced search and query commands for finding heroes, clans, and kingdoms with flexible filtering options. These commands support both text search and type-based filtering with AND/OR logic capabilities.

Tip: Use query commands before management commands to find exact IDs and ensure you're targeting the correct entities. Query results display StringIds which can be used in other commands.

Hero Query Commands

gm.query.hero SEARCH
Search for heroes using text search and type filters with AND logic (all criteria must match). Supports flexible combinations of search terms and type keywords.

Parameters

[search terms] string Optional
Text to search in hero names, IDs, clan names, and kingdom names
[type keywords] keywords Optional
Type filters: lord, wanderer, notable, merchant, child, female, male, clanleader, kingdomruler, partyleader, fugitive, alive, dead, prisoner, withoutclan, withoutkingdom, married

Examples

Find female lords in Empire
gm.query.hero empire lord female
Find all wanderers
gm.query.hero wanderer
Find dead kingdom rulers
gm.query.hero dead kingdomruler
Search by name
gm.query.hero john

Returns

Success: "Found {count} hero(es) matching {criteria}:"
Lists heroes with: Name, StringId, Clan, Kingdom, Age, Status (Alive/Dead/Prisoner)
Note: Without type keywords specifying life status, the command defaults to showing only living heroes. Use dead keyword to search deceased heroes.
gm.query.hero_any SEARCH
Search for heroes using OR logic (matches ANY of the specified criteria instead of ALL). Useful for broader searches across multiple types.

Parameters

[search terms] string Optional
Text to search in hero names and identifiers
[type keywords] keywords Optional
Same type keywords as gm.query.hero, but uses OR logic

Examples

Find anyone who is a lord OR wanderer
gm.query.hero_any lord wanderer
Find clan leaders OR kingdom rulers
gm.query.hero_any clanleader kingdomruler

Returns

Success: "Found {count} hero(es) matching ANY of {criteria}:"
Lists matching heroes with full details

Search Logic Explained

AND Logic (Default): When using standard query commands (gm.query.hero, gm.query.clan, gm.query.kingdom), ALL specified criteria must match. For example, gm.query.hero empire lord female finds heroes who are Empire AND lord AND female.
OR Logic: When using _any variants (gm.query.hero_any, etc.), results match ANY of the specified type criteria. For example, gm.query.hero_any lord wanderer finds heroes who are EITHER lords OR wanderers.
