Hero Management Commands

Commands for managing individual heroes in Bannerlord. These commands allow you to modify hero properties, transfer heroes between clans, manage their state (alive/dead/prisoner), and set relationships.

Note: All hero commands require you to be in campaign mode. Many operations are permanent and cannot be undone, so use with caution and consider backing up your save first.

Clan Management

gm.hero.set_clan CLAN
Transfers a hero from their current clan to another clan.

Parameters

hero string Required
Hero identifier (StringId or partial name match)
clan string Required
Target clan identifier (StringId or partial name match)

Examples

Transfer a hero by StringId
gm.hero.set_clan lord_1_1 clan_empire_south_1
Transfer using partial names
gm.hero.set_clan aserai empire

Returns

Success: "{Hero Name} (ID: {hero_id}) transferred from '{Previous Clan}' to '{New Clan}'."
Includes updated hero details with new clan information.
gm.hero.remove_clan CLAN
Removes a hero from their current clan, leaving them without a clan affiliation.

Parameters

hero string Required
Hero identifier (StringId or partial name match)

Examples

Remove a hero from their clan
gm.hero.remove_clan lord_1_1

Returns

Success: "{Hero Name} (ID: {hero_id}) removed from clan '{Clan Name}'."
Note: Hero must currently be a member of a clan. Returns an error if the hero has no clan.

State Management

gm.hero.kill STATE
Kills the specified hero. This is a permanent action that cannot be undone.

Parameters

hero string Required
Hero identifier (StringId or partial name match)
show_death_log boolean Optional
If true, displays the death log in-game (default: false)

Examples

Kill a hero without death log
gm.hero.kill lord_1_1
Kill a hero and show death log
gm.hero.kill lord_1_1 true

Returns

Success: "{Hero Name} (ID: {hero_id}) has been killed."
Warning: This action is permanent and cannot be undone. The hero will be marked as dead and removed from active gameplay. Consider backing up your save before using this command.
gm.hero.imprison STATE
Imprisons a hero, making them a prisoner of another hero or party.

Parameters

prisoner string Required
Hero to be imprisoned (StringId or partial name match)
captor string Required
Hero who will hold the prisoner (StringId or partial name match)

Examples

Imprison one hero by another
gm.hero.imprison lord_1_1 lord_2_1

Returns

Success: "{Prisoner Name} (ID: {prisoner_id}) is now imprisoned by {Captor Name}."
Note: The captor must have a valid party or settlement to hold prisoners. Returns an error if the hero is already a prisoner or if the captor cannot hold prisoners.
gm.hero.release STATE
Releases a hero from captivity, making them free again.

Parameters

hero string Required
Imprisoned hero identifier (StringId or partial name match)

Examples

Release a hero from captivity
gm.hero.release lord_1_1

Returns

Success: "{Hero Name} (ID: {hero_id}) has been released from captivity."
Note: Hero must currently be a prisoner. Returns an error if the hero is not imprisoned.

Attribute Management

gm.hero.set_age ATTRIBUTE
Sets the hero's age to a specific value.

Parameters

hero string Required
Hero identifier (StringId or partial name match)
age float Required
New age value (range: 0-128)

Examples

Set hero age to 30
gm.hero.set_age lord_1_1 30
Set hero age with decimal precision
gm.hero.set_age wanderer_1 25.5

Returns

Success: "{Hero Name}'s age changed from {old_age} to {new_age}."
gm.hero.set_gold ATTRIBUTE
Sets the hero's gold amount to a specific value.

Parameters

hero string Required
Hero identifier (StringId or partial name match)
amount integer Required
New gold amount

Examples

Set hero gold to 10,000
gm.hero.set_gold lord_1_1 10000
Set hero gold to zero
gm.hero.set_gold merchant_1 0

Returns

Success: "{Hero Name}'s gold changed from {old_gold} to {new_gold}."
gm.hero.add_gold ATTRIBUTE
Adds or subtracts gold from a hero's current amount. Use negative values to subtract gold.

Parameters

hero string Required
Hero identifier (StringId or partial name match)
amount integer Required
Amount to add (positive) or subtract (negative)

Examples

Add 5,000 gold to hero
gm.hero.add_gold lord_1_1 5000
Subtract 1,000 gold from hero
gm.hero.add_gold merchant_1 -1000

Returns

Success: "{Hero Name}'s gold changed from {old_gold} to {new_gold} (+{amount})."
gm.hero.heal ATTRIBUTE
Instantly heals a hero to full health, setting their hit points to maximum.

Parameters

hero string Required
Hero identifier (StringId or partial name match)

Examples

Heal a wounded hero
gm.hero.heal lord_1_1

Returns

Success: "{Hero Name} has been healed to full health ({current_hp}/{max_hp})."
Note: Cannot heal dead heroes. Hero must be alive to use this command.

Relationship Management

gm.hero.set_relation RELATION
Sets the relationship value between two heroes. Relations range from -100 (enemies) to 100 (close friends).

Parameters

hero1 string Required
First hero identifier (StringId or partial name match)
hero2 string Required
Second hero identifier (StringId or partial name match)
value integer Required
Relationship value (range: -100 to 100)

Examples

Set two heroes as friends
gm.hero.set_relation lord_1_1 lord_2_1 50
Set two heroes as enemies
gm.hero.set_relation lord_1_1 lord_3_1 -75
Set neutral relation
gm.hero.set_relation wanderer_1 notable_1 0

Returns

Success: "Relation between {Hero1 Name} and {Hero2 Name} changed from {old_relation} to {new_value}."
Note: Relationship values affect diplomacy, marriage eligibility, and hero interactions. Values below -10 indicate dislike, while values above 10 indicate friendship.

gm.hero.remove_clan - Remove hero from clan
State Management:
gm.hero.kill - Kill hero
gm.hero.imprison - Imprison hero
gm.hero.release - Release from prison
Attributes:
gm.hero.set_age - Set hero age
gm.hero.set_gold - Set hero gold amount
gm.hero.add_gold - Add/subtract gold
gm.hero.heal - Heal to full health
Relationships:
gm.hero.set_relation - Set relation between heroes
